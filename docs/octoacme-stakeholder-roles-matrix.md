# OctoAcme — Stakeholder Roles Matrix

## Purpose
This template clarifies which roles participate in each phase of an OctoAcme project. Use it at project kickoff to ensure all key contributors are identified and engaged at the right times.

## How to Use
1. Copy this template into your project documentation folder.
2. Fill in the named individuals for each role in your project.
3. Update the participation level for each phase as needed.

## Participation Levels
| Symbol | Meaning |
|--------|---------|
| **L** | Lead — owns this activity and drives decisions |
| **C** | Contributor — actively participates and provides input |
| **R** | Reviewer — reviews outputs and provides feedback |
| **I** | Informed — receives updates but does not actively participate |
| _(blank)_ | Not involved in this phase |

## Phase Descriptions

### Initiation
Define the problem statement, identify stakeholders, establish high-level goals and success criteria, and confirm project sponsorship.

### Planning
Develop the project plan, define scope and milestones, identify risks, assign resources, and finalize requirements and designs.

### Execution
Build, test, and iterate on deliverables. Track progress, manage blockers, and conduct regular reviews and demos.

### Release
Validate release readiness, deploy to production, communicate release details, and confirm post-deploy verification.

### Close & Retrospective
Capture final outcomes, complete handoff documentation, conduct retrospectives, and archive project artifacts.
